# lost-periods
Challenge #1 of Previred assessment: SpringBoot application to find lost periods between two random distinct dates

Se recomienda utilizar el IDE Spring Tool Suite para editar. Importar como proyecto Gradle.

*Ejecutar las siguientes instrucciones para compilar, empaquetar JAR y ejecutar:

# Compilacion. Abrir una shell y posicionarse dentro del directorio "lost-periods" (carpeta raíz) y ejecutar el siguiente comandos según SO:

*En UNIX
    ./gradlew build
    
*En Windows
    gradlew.bat build
  

# Ejecución de la aplicación: para levantar la API REST de Lost Periods ejecutar el siguiente comando desde el mismo directorio raíz:

java -jar build/libs/lost-periods-api-1.0.0.jar

La solucion disponibiliza la siguiente operación:


* Cálculo de Períodos Perdidos

URL: http://127.0.0.1:8081/lost-periods/api/v1/list
Tipo de Operación: GET

Esta operación invoca al servicio REST de GDD suministrado por Previred y con el conjunto de datos aleatorios obtenidos, calcula
los períodos faltantes entre las 2 fechas indicadas, que no se encuentren en la lista de fechas.
La aplicación requiere se configuren los siguientes parámetros que se encuentra en el archivo "src/main/resources/application.properties"
en la raíz del proyecto:

server.port=8081  (puerto en el que se levanta la API)
com.previred.lost.periods.generador_datos_desafio.service_protocol=http://    [protocolo en el que se encuentra desplegado servicio GDD]
com.previred.lost.periods.generador_datos_desafio.service_host=127.0.0.1    [hostname o IP en el que se encuentra desplegado servicio GDD]
com.previred.lost.periods.generador_datos_desafio.service_port=8080   [puerto en el que se encuentra desplegado servicio GDD]
com.previred.lost.periods.generador_datos_desafio.context_path=/periodos/api  [contexto de la operación del servicio GDD que genera los períodos aleatorios]

El Swagger de la API se puede encontrar en la siguiente URL, una vez levantada la API:

http://127.0.0.1:8081/swagger-ui.html#!/lost32periods/lost_periods

*Request:
No recibe parámetros ni un JSON de entrada

*Response:
Retorna la misma información recibida del servicio GDD (conjunto de datos) encapsulándolos dentro de un objeto llamado "lostPeriods" y
adicionando el elemento al JSON llamado "fechasFaltantes", que contiene la lista de los períodos perdidos calculados en base a dicho conjunto
de datos.
Adicionalmente, se incorpora un objeto llamado "status" que agrupa la información del resultado de la ejecución de la operación: código HTTP,
descripción del código HTTP una descripción personalizada del resultado.
